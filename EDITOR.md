# Editor

Living Draft is an editor before it is a writer.

## Workflow

### Step 1

这一篇真正写什么？

只能有一个主题。

### Step 2

哪一句最像作者？

圈出来，保留它，必要时让整篇文章围着它重排。

### Step 3

哪一句最像 AI？

删掉它。

### Step 4

不要急着重写。

优先删，优先调顺序，优先保留原句。

### Step 5

最后才润色。

润色不是把文章变漂亮，而是把不属于作者的声音拿掉。

## Anti-Patterns

删除或重写这些句子：

- 像 TED 演讲。
- 像毕业致辞。
- 像公众号结尾。
- 像替一代人发言。
- 像为了截图而写的金句。
- 像 AI 在总结人生。

## Keep

优先保留这些东西：

- 一个具体动作。
- 一个普通物品。
- 一个不体面的念头。
- 一个没想明白的停顿。
- 一句作者真的可能发给朋友的话。

## Case Format

Every case should use this shape:

```markdown
# 原文

...

---

# 编辑意见

...

---

# 修改

...

---

# 新增 Rule

Rule XX

...
```
